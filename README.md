# Alpha Extension

## 使用方法（给交易者，无需技术背景）

### 第一步：打开插件

1. 点浏览器右上角的 **拼图图标**，找到 **Alpha Extension**。  
2. 点开小窗口，**勾选开关**，交易页面上才会出现 **下单面板**。关掉开关，面板会隐藏。  
3. 若正在自动买卖时关掉开关，会先**停止当前任务**再收起面板。

### 第二步：进入交易页

用 **Chrome** 打开 **bbn Alpha** 交易页面（插件支持的网址），等网页加载完。稍等片刻，页面 **左上角** 会出现可拖动的 **下单面板**。

### 第三步：看懂面板里每一项

| 区域 | 做什么 |
|------|--------|
| **合约标记价 / 最新价** | 仅供参考，方便对行情。 |
| **交易方向** | 选 **买入** 或 **卖出**（和你在页面上买卖的方向一致）。 |
| **交易模式** | 目前请使用 **「目标价交易模式」**（默认）。 |
| **目标价** | **必填**。只有盘口价格满足条件时才会自动去点下单（例如买入时：卖一价不高于你填的目标价）。必须是 **大于 0 的数字**。 |
| **总限制（选填）** | 控制**整段自动任务里**累计成交金额或累计数量，避免跑过头。右侧可选 **USDT** 或 **数量**。不填表示不设总上限。 |
| **单次限制（选填）** | 控制**每一笔**最多用多少 **USDT** 或多少 **标的数量**。不填时，单笔大致按订单簿上你点的那一档来。右侧可选单位。 |
| **不产生委托单 / 允许产生委托单** | **默认是不勾选**（显示「不产生委托单」）：若你填的单次限制 **大于** 订单簿上这一档显示的量，插件会把单笔 **压到**「单次限制」和「该档盘口」里**较小的那个**，尽量**避免出现未吃完的挂单**。若勾选「允许产生委托单」，则按你填的单次限制下单，超出该档的部分有可能变成**挂单**（以交易所实际规则为准）。 |
| **交易时间间隔（选填）** | 填 **秒数**（可填小数，例如 `2` 表示 2 秒）。**上一笔在页面上确认完成之后**，再等这段时间，才会尝试下一笔。不填则**不额外等待**（仍会有页面确认等必要间隔）。 |
| **自动买入 / 自动卖出** | 点一次开始循环；运行中按钮会显示 **进行中**，此时不要重复点。 |
| **停止** | 随时停止自动循环。 |
| **状态与操作日志** | 上面一行是当前提示；下面是带时间的**操作记录**，方便你回看。 |

### 第四步：典型操作流程

1. 选好 **买入** 或 **卖出**。  
2. 填好 **目标价**（以及需要的 **总限制 / 单次限制**）。  
3. 根据需要勾选是否 **允许产生委托单**，或填 **交易时间间隔**。  
4. 点 **「自动买入」** 或 **「自动卖出」**。  
5. 若页面弹出确认框，插件会尝试帮你点 **继续 / 确认**；你也可以自己点。  
6. 想停就点 **「停止」**。

### 切换交易币种时

当你在同一浏览器里 **换成另一个币** 时，面板会**跟着更新**币种和行情，并**清空**目标价、总限制、单次限制、时间间隔等（**不会清空操作日志**），避免用旧参数误操作新币种。

### 自动运行中

**目标价、总限制、单次限制、委托单选项、时间间隔** 会锁死，不能改。先点 **「停止」**，改完再重新开始。

## 安装插件（第一次使用，可请身边懂电脑的人帮忙）

1. 打开 Chrome，地址栏输入 **`chrome://extensions`** 回车。  
2. 打开右上角的 **「开发者模式」**。  
3. 点 **「加载已解压的扩展程序」**。  
4. 选中本仓库的**根目录**（即包含 **`manifest.json`** 的那一层文件夹；若从 Git 克隆，默认文件夹名一般为 **`token-trade-tools`**）。

装好后，按上面 **「使用方法」** 操作即可。  
若改动了插件里的文件，需要在本扩展那一行点 **「更新」**，并**刷新**交易网页，改动才会生效。

## 给开发者的说明

本仓库即完整扩展源码：**`manifest.json`** 在根目录，业务脚本在 **`src/`**（后台、弹窗、选项页、各 content 脚本）。

当前内容脚本匹配 **binance.com** 的 Alpha 路径。若需改匹配范围，请编辑 **`manifest.json`** 中 `content_scripts` 的 `matches` 字段。
